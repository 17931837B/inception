# Inception - User Documentation

このドキュメントは、Inceptionプロジェクトで構築されたシステムのエンドユーザーおよび管理者向けの基本的な使用手順を説明するものです。

## 1. スタックアーキテクチャの概要
本システムは、Docker Composeを用いて構築された以下の3つの独立したコンテナで構成されています。
* **NGINX**: TLSv1.2 / TLSv1.3を利用したHTTPS専用のWebサーバー（ポート443経由でのみアクセス可能）。
* **WordPress + php-fpm**: サイトのコンテンツを処理・生成するアプリケーションサーバー。
* **MariaDB**: WordPressのデータを安全に保存するリレーショナルデータベース。

## 2. サイトおよび管理パネルへのアクセス
お使いのWebブラウザから、以下のURLにアクセスしてシステムを利用します。

* **ウェブサイト（公開ページ）**: `https://tobaba.42.fr`
* **管理パネル（ダッシュボード）**: `https://tobaba.42.fr/wp-admin`

※ 注意: 本システムは自己署名証明書を使用しているため、ブラウザによっては初回アクセス時にセキュリティ警告が表示されます。これは安全なローカル環境での想定された挙動ですので、警告をスキップしてアクセスを継続してください。

## 3. 資格情報の管理
システムには初期状態で以下の2つのアカウントが設定されており、パスワードなどの認証情報は `.env` ファイルで管理されています。

* **管理者アカウント**: サイトの全体設定やページ編集を行う権限を持ちます（セキュリティ要件により、ユーザー名に「admin」は含まれません）。
* **一般ユーザーアカウント**: 記事へのコメント投稿などが可能な権限が制限されたアカウントです。

**パスワードの変更手順:**
1. 管理パネル (`/wp-admin`) にログインします。
2. 左側メニューから「ユーザー」 > 「プロフィール」を開きます。
3. 画面下部の「アカウント管理」セクションにある「新しいパスワードを設定」から任意のパスワードに変更し、「プロフィールを更新」をクリックします。

## 4. 基本的なチェック
システムが正常に動作していることを確認するため、定期的に以下のテストを実施してください。

* **HTTPアクセスのブロック**: `http://tobaba.42.fr` (ポート80) にアクセスし、接続が拒否されることを確認する。
* **コメント機能の確認**: 一般ユーザーアカウントでログインし、記事にコメントを追加して画面に反映されるか確認する。
* **コンテンツ編集の確認**: 管理者アカウントでログインし、ダッシュボードからページを編集して、ウェブサイト上で変更が正しく反映されているか確認する。
