# Inception - User Documentation

[cite_start]このドキュメントは、Inceptionプロジェクトで構築されたシステムのエンドユーザーおよび管理者向けの基本的な使用手順を説明するものです [cite: 83, 422, 423]。

## 1. スタックアーキテクチャの概要
[cite_start]本システムは、Docker Composeを用いて構築された以下の3つの独立したコンテナで構成されています [cite: 269, 270]。
* [cite_start]**NGINX**: TLSv1.2 / TLSv1.3を利用したHTTPS専用のWebサーバー（ポート443経由でのみアクセス可能） [cite: 280, 315]。
* [cite_start]**WordPress + php-fpm**: サイトのコンテンツを処理・生成するアプリケーションサーバー [cite: 281]。
* [cite_start]**MariaDB**: WordPressのデータを安全に保存するリレーショナルデータベース [cite: 282]。

## 2. プロジェクトの起動と停止
[cite_start]本システムのコンテナ群を開始および停止するには、リポジトリのルートディレクトリで以下のコマンドを実行します 。

* **プロジェクトの開始**: `make all` （または `make`）を実行します。システムがバックグラウンドでビルドおよび起動します。
* **プロジェクトの停止**: `make down` を実行します。保存されたデータ（データベースやサイトファイル）を保持したまま、安全にシステムを停止します。

## 3. サイトおよび管理パネルへのアクセス
[cite_start]お使いのWebブラウザから、以下のURLにアクセスしてシステムを利用します [cite: 426]。

* **ウェブサイト（公開ページ）**: `https://tobaba.42.fr`
* **管理パネル（ダッシュボード）**: `https://tobaba.42.fr/wp-admin`

[cite_start]※ 注意: 本システムは自己署名証明書を使用しているため、ブラウザによっては初回アクセス時にセキュリティ警告が表示されます。これは安全なローカル環境での想定された挙動ですので、警告をスキップしてアクセスを継続してください [cite: 114]。

## 4. 資格情報の管理
[cite_start]システムには初期状態で以下の2つのアカウントが設定されており、パスワードなどの認証情報は環境変数 (`.env` ファイル) 経由で安全に管理されています [cite: 311, 427]。

* [cite_start]**管理者アカウント**: サイトの全体設定やページ編集を行う権限を持ちます（セキュリティ要件により、ユーザー名に「admin」や「administrator」等は含まれません） [cite: 126, 302]。
* [cite_start]**一般ユーザーアカウント**: 記事へのコメント投稿などが可能な、権限が制限されたアカウントです [cite: 124]。

**パスワードの変更手順:**
1. 管理パネル (`/wp-admin`) にログインします。
2. 左側メニューから「ユーザー」 > 「プロフィール」を開きます。
3. 画面下部の「アカウント管理」セクションにある「新しいパスワードを設定」から任意のパスワードに変更し、「プロフィールを更新」をクリックします。

## 5. 基本的なチェック
[cite_start]システムが正常に動作していることを確認するため、定期的に以下のテストを実施してください [cite: 83, 428]。

* [cite_start]**HTTPアクセスのブロック**: `http://tobaba.42.fr` (ポート80) にアクセスし、接続ができないことを確認する [cite: 111]。
* [cite_start]**コメント機能の確認**: 一般ユーザーアカウントでログインし、記事にコメントを追加して画面に反映されるか確認する [cite: 124]。
* [cite_start]**コンテンツ編集の確認**: 管理者アカウントでログインし、ダッシュボードからページを編集して、ウェブサイト上で変更が正しく反映されているか確認する [cite: 126]。
